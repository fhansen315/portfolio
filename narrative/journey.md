# The Journey: SDR to Automation Engineer

> How I taught myself to code and built systems that transformed sales productivity

## The Starting Point

**Role**: Sales Development Representative at Ripple
**Location**: Brazil/LATAM Market
**Year**: 2024

I joined Ripple as an SDR, excited to work in the crypto/fintech space. The role was traditional—research prospects, send emails, book meetings, repeat. What I didn't expect was how much of my day would be consumed by repetitive, manual tasks.

## The Breaking Point

After my first month, I did the math:

| Manual Task | Hours per Campaign |
|-------------|-------------------|
| Lead research | 63.5 hours |
| Email personalization | 21 hours |
| Lead scoring | 10.5 hours |
| Data entry & updates | 5+ hours |
| **Total** | **95+ hours** |

That's over two weeks of work for a single campaign targeting ~127 leads.

I thought: *"There has to be a better way."*

## The Decision

Instead of accepting this as "just how sales works," I made a decision that changed my career trajectory:

**I would learn to code and automate my own job.**

Not to eliminate it—but to transform it. To spend less time on repetitive tasks and more time on what actually matters: building relationships and closing deals.

## The Learning Curve

### Phase 1: Python Basics (Months 1-2)
- Online courses and tutorials
- Small scripts to parse CSV files
- First automations: formatting data, generating reports

### Phase 2: API Integration (Months 3-4)
- Learned HTTP, REST, JSON
- Connected to Salesforce CLI
- Built first data extraction pipelines

### Phase 3: AI/ML (Months 5-6)
- Discovered LangChain and CrewAI
- Built multi-agent research systems
- Integrated LLMs for content generation

### Phase 4: Full Stack (Months 7-12)
- Added TypeScript for event-driven systems
- Built React dashboards for visualization
- Deployed containerized applications

## The Insight

The most valuable thing I learned wasn't any specific technology—it was this:

> **The best automation doesn't replace human judgment. It amplifies it.**

My systems handle the 80% of work that's repetitive and rules-based. This frees me (and other SDRs) to focus on the 20% that requires creativity, empathy, and strategic thinking.

## Why Automation Engineering?

After a year of building these systems while exceeding my SDR targets, the path forward became clear:

I want to do this full-time.

Not just automate my own workflows—but build automation systems for entire organizations. The impact scales exponentially when you're building tools that help teams, not just individuals.
